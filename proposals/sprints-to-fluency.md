## Overview

The "Sprints to Fluency" plan structures vocabulary acquisition into **19 two-week sprints** across 9 months. Each sprint follows a "Learn-Apply-Audit" rhythm:

```
Sprint (2 weeks)
├── Week 1: Acquisition & Input — learn ~265 new words (19-20/day)
└── Week 2: Application & Output — quiz, write, speak using those words
```

### Targets

| Period | Target |
|--------|--------|
| Per day | 19–20 new words |
| Per week | ~132 new words |
| Per sprint (2 weeks) | ~265 new words |
| Per quarter | ~1,600 words |
| Total by year end | 5,000 words |

## 2. Sprint System (Core)

The Sprint system is the backbone of the entire plan. Every other feature depends on it.

### Data Model

```
Sprint
├── id                  String (cuid)
├── userId              String
├── number              Int          // Sprint #1, #2, ... #19
├── status              Enum         // PLANNED | ACTIVE | COMPLETED | REVIEW
├── startDate           DateTime
├── endDate             DateTime
├── wordTarget          Int          // Default: 265
├── wordsLearned        Int          // Computed from WordProgress
├── reviewTarget        Float        // Default: 0.85 (85% retention)
├── retentionRate       Float?       // Computed after Week 2 quiz
├── isReviewSprint      Boolean      // Every 4th sprint = 50% review, 50% new
├── phase               Enum         // ACQUISITION | APPLICATION
├── createdAt           DateTime
└── updatedAt           DateTime

SprintWord
├── id                  String (cuid)
├── sprintId            String
├── wordId              String
├── status              Enum         // NEW | LEARNING | MASTERED
├── sentences           Json?        // User-written sentences for this word
└── quizzed             Boolean      // Whether tested in Week 2 quiz
```

### Sprint Lifecycle

```
PLANNED → ACTIVE → (auto week 2) → COMPLETED
              ↑                         │
              └──── REVIEW sprint ──────┘  (every 4th sprint)
```

### Pace Calculator Logic

```typescript
function calculatePace(
  wordsLearned: number,
  totalTarget: number,
  startDate: Date,
  deadline: Date,
  now: Date
): PaceResult {
  const daysElapsed = daysBetween(startDate, now)
  const daysRemaining = daysBetween(now, deadline)
  const wordsPerDayActual = wordsLearned / daysElapsed
  const wordsPerDayNeeded = (totalTarget - wordsLearned) / daysRemaining
  const projectedTotal = wordsLearned + (wordsPerDayActual * daysRemaining)
  
  return {
    wordsPerDayActual,      // "14 words/day"
    wordsPerDayNeeded,      // "13 words/day"
    onTrack: projectedTotal >= totalTarget,
    projectedCompletion: ...,  // Estimated date to hit target
    projectedTotal,           // Projected total words by deadline
  }
}
```

## 6. Strategic Success Features

### What the plan recommends

1. **Goldilocks Difficulty**: Use frequency lists, don't pick random words
2. **Review Buffer**: Every 4th sprint, reduce new words by 50%, review previous 3 sprints
3. **Write for Real**: Context-based writing (reviews, emails, stories)
4. **Niche Pivot**: If plateau in Q3, switch to themed learning (tech, psychology, cooking)

### What we have

| Feature | Status | Details |
|---------|--------|---------|
| Frequency-based word ordering | ✅ Done | Words sorted by `frequency` field |
| Oxford 5000 frequency list | ✅ Done | High-quality source data |
| Theme categories | ✅ Done | 10 themes for niche-based learning |
| Spaced repetition | ✅ Done | Words reviewed at optimal intervals |
| Study lists | ✅ Done | Custom lists for themed study |
| Quiz from list | ✅ Done | Can quiz on specific word sets |

### What we need

| Feature | Priority | Effort | Description |
|---------|----------|--------|-------------|
| **Review sprint (every 4th)** | 🟡 P1 | Small | Auto-flag every 4th sprint as review sprint: 50% new, 50% review words from last 3 sprints |
| **Review sprint word selection** | 🟡 P1 | Medium | Collect weak words (low ease factor, missed in quizzes) from last 3 sprints |
| **Plateau detection** | 🟢 P2 | Medium | Detect when retention rate drops or XP gain plateaus, suggest pivot |
| **Niche/theme suggestions** | 🟢 P2 | Small | Based on quarter focus + plateau, suggest theme to study |
| **Context-aware writing prompts** | 🟢 P2 | Medium | Generate writing prompts using sprint words (e.g., "Write a product review using these 10 words") |

### Review Sprint Logic

```
Sprint #1  →  Sprint #2  →  Sprint #3  →  Sprint #4 (REVIEW)
  265 new      265 new       265 new       130 new + 135 review
                                            ↑ from Sprints #1-3
                                            (words with lowest retention)
```

## Appendix: Data Model Additions

### New Tables

```prisma
model Sprint {
  id            String   @id @default(cuid())
  userId        String
  number        Int
  status        SprintStatus @default(PLANNED)
  startDate     DateTime
  endDate       DateTime
  wordTarget    Int      @default(265)
  reviewTarget  Float    @default(0.85)
  isReviewSprint Boolean @default(false)
  phase         SprintPhase @default(ACQUISITION)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  user          User     @relation(fields: [userId], references: [id])
  words         SprintWord[]
  writings      SprintWriting[]

  @@unique([userId, number])
}

model SprintWord {
  id          String   @id @default(cuid())
  sprintId    String
  wordId      String
  sentences   Json?
  quizzed     Boolean  @default(false)
  quizCorrect Boolean  @default(false)

  sprint      Sprint   @relation(fields: [sprintId], references: [id])
  word        Word     @relation(fields: [wordId], references: [id])

  @@unique([sprintId, wordId])
}

model SprintWriting {
  id          String   @id @default(cuid())
  sprintId    String
  type        WritingType  // SENTENCE | LONG_FORM
  text        String
  wordCount   Int
  sprintWordsUsed Int
  feedback    Json?        // LLM feedback (future)
  createdAt   DateTime @default(now())

  sprint      Sprint   @relation(fields: [sprintId], references: [id])
}

model Milestone {
  id          String   @id @default(cuid())
  userId      String
  name        String
  wordTarget  Int
  deadline    DateTime
  focusArea   String?      // "academic verbs", "idioms", etc.
  createdAt   DateTime @default(now())

  user        User     @relation(fields: [userId], references: [id])
}

enum SprintStatus {
  PLANNED
  ACTIVE
  COMPLETED
}

enum SprintPhase {
  ACQUISITION    // Week 1: learn new words
  APPLICATION    // Week 2: quiz, write, speak
}

enum WritingType {
  SENTENCE       // Short sentence for a single word
  LONG_FORM      // 500+ word challenge
}
```
